## 03. お悩みセクション「こんなお悩みありませんか？」

### 見出し
```
楽天ふるさと納税、
こんな課題に直面していませんか？
```

### 悩みカード（6項目）
各カードは「アイコン＋悩み本文＋補足説明」構成。

1. **寄付額が伸び悩んでいる**
   - 前年対比で伸びが鈍化。競合自治体との差が広がっている。
2. **RPP広告のROASが低迷している**
   - 広告費は増えるのに寄付額が比例しない。採算が取れない。
3. **9月・12月のピーク期に対応しきれない**
   - レビュー規制強化以降、集中投資の戦略が組めていない。
4. **ランキング競争に勝てない**
   - 上位掲載されず、新規寄付者に見つけてもらえない。
5. **返礼品が多すぎて優先順位がつかない**
   - どのSKUに広告投資を集中すべきか判断できない。
6. **社内に楽天運用の専門人材がいない**
   - 担当者が他業務と兼任。ノウハウが属人化している。

### 締めコピー
```
一つでも当てはまるなら、運用の「型」を変える時です。
```

## 05. CARDSが選ばれる6つの理由

### 見出し
```
CARDSが選ばれる、6つの理由。
```

### 6つの強みカード（3列×2行）

#### 01. 楽天ふるさと納税への専門特化
ふるさと納税市場の特殊性（季節変動・規制・プラットフォーム仕様）を熟知。
「楽天モール運用」と「ふるさと納税運用」の両方に精通した稀有な支援会社です。

#### 02. ROAS 700%〜2,000%超の実績
支援自治体の全体ROAS 1,358%、個別キャンペーンでは2,000%超を達成。
前年対比寄付額150%を実現した自治体も複数あります。

#### 03. 「寄付額最大化」を最上位KPIに設定
ROASを追うだけでは寄付総額は伸びません。CARDSは広告費対効果と寄付額伸長の
両立を設計し、ビッグキーワード露出率85〜90%を維持しながらROASも最適化します。

#### 04. 複合キーワード戦略
「ふるさと納税 いくら」等の指名キーワードに加え、「いくら」「サーモン」等の
単体キーワードを並行運用。一般的な運用の倍近い露出機会を創出します。

#### 05. 消費財EC運用ノウハウの転用
楽天モール月商1,000万〜2,500万円クラスの実戦経験に基づく商品ページ最適化、
クリエイティブ制作、キーワード分析。机上の空論ではない運用を提供します。

#### 06. 市場変化への機動的対応
レビュー規制強化、楽天仕様変更、ランキング競争激化——。
変化の早いふるさと納税市場に、週次モニタリング＋月次定例で即応します。

## 12. FAQ（10項目）

### 見出し
```
よくあるご質問。
```

### アコーディオン形式

**Q1. 最低契約期間はありますか？**
A. 原則6ヶ月を推奨しております。ふるさと納税は季節変動が大きく、年間を通した設計が成果最大化に繋がるためです。短期の試験運用も個別にご相談ください。

**Q2. 成果が出るまでどれくらいかかりますか？**
A. 運用開始から1〜2ヶ月で初期改善（キーワード最適化・クリエイティブ改善によるROAS改善）、3〜6ヶ月で寄付額30〜50%増を実現する事例が多いです。

**Q3. さとふる・ふるなび等の他プラットフォームにも対応できますか？**
A. 現在は楽天ふるさと納税に特化していますが、他プラットフォームを含めた総合戦略のご相談は承っております。

**Q4. 小規模自治体でも依頼可能ですか？**
A. 可能です。広告予算規模に応じた柔軟なプラン設計を行います。

**Q5. 広告予算の最低ラインはありますか？**
A. 月額広告予算の下限は個別にご相談となります。予算規模に応じた最適な戦略をご提案します。

**Q6. レポーティングの頻度は？**
A. 週次で広告メモ（ROAS・寄付額・キーワード分析）を共有し、月次で定例会議を実施します。

**Q7. 自治体直接契約／DMO経由契約、どちらも可能ですか？**
A. どちらにも対応可能です。既存のDMO・パートナーコンサル経由での受託実績も複数あります。

**Q8. 返礼品事業者からの相談も受けられますか？**
A. はい。自治体案件に加え、返礼品提供事業者からのご相談にも対応しています。

**Q9. 既存の広告代行業者からの切り替えは可能ですか？**
A. 可能です。引き継ぎ時のデータ移行・運用移管もサポートいたします。

**Q10. レビューキャンペーン規制への対応はどうしていますか？**
A. 規制強化以降、9月・12月のピーク期への集中投資戦略に切り替え、ROAS維持と寄付額最大化を両立しています。

## 実装上のメモ

- 全セクションに適切なpadding（PC: 120px上下／SP: 64px上下）
- セクション間の区切りは背景色の切り替え（白⇔淡いグレー⇔淡いネイビー）で演出
- CTAボタンは右下にフローティング固定表示（スクロール時も常時見える）
- 各セクション見出しは大きめ（desktop: 40-48px、SP: 28-32px）
- 数字（ROAS 1,358%等）は特大フォントでインパクトを持たせる
- 参考LPと同水準の「重厚感・信頼感」を優先し、情報量を削らない
